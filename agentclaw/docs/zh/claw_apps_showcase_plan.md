# AgentClaw 内置趣味工作流与 Claw Apps 模板库方案

## 背景

AgentClaw 现在的能力已经比较丰富：声明式工作流、Agent Creator、Skills、MCP、知识库、文件处理、浏览器/电脑操作、调试追踪、定时任务、渠道接入和图片生成等都已经具备。但这些能力在对外展示时偏工程化，非开发人员很难在短时间内理解“我为什么要用它”或“这个项目有什么好玩之处”。

GitHub Star 往往不只来自严肃的技术评估，也来自第一眼的兴趣、收藏欲和传播感。AgentClaw 如果只展示框架能力，会吸引框架型开发者；如果增加一层可直接体验、可截图、可分享的内置应用，就能让普通用户、产品经理、创作者和 AI 爱好者也产生兴趣。

因此，可以在 AgentClaw 中增加一组官方趣味工作流模板，并通过“模板库”入口发现。这些模板既是可玩的应用，也是最佳实践示例，还是对外传播素材。但它们不应默认混入用户项目的智能体列表；只有用户明确“导入智能体”后，才成为当前项目里的普通工作流。

## 核心判断

这个方向值得做，但不应该被定位为“示例工作流列表”。“示例”这个词会让用户觉得它只是文档附属品，主要服务开发者学习；更好的定位是：

> AgentClaw 自带一组可一键导入为智能体、可查看实现、可复制改造的 Claw Apps 模板。

这句话同时服务三类人：

- 非开发者：可以在模板库里看到玩法，再一键导入为自己的智能体体验。
- 开发者：可以查看工作流实现，学习 AgentClaw 的真实用法。
- 二次开发者：可以复制为自己的工作流，继续改造成业务应用。

## 产品目标

1. 提升 GitHub 首屏吸引力，让访问者在不理解框架细节前先看到有趣结果。
2. 降低新用户理解成本，用具体应用解释 AgentClaw 的能力边界。
3. 提供高质量官方模板，让开发者从“能跑的应用”开始学习，而不是从抽象 API 开始。
4. 产出可用于 README、官网、社媒、短视频和 GIF 的演示素材。
5. 建立一个未来可以扩展为模板市场、能力库或工作流社区的基础形态。

## 入口形态

第一版应新增独立导航入口，而不是把趣味工作流直接塞进 `智能体` 或 `内置智能体`：

- `智能体`：只展示当前项目已经拥有、已经加载、可直接运行的工作流。
- `内置智能体`：只保留 AgentClaw 系统通用助手，不混入玩法型应用。
- `模板库`：展示 AgentClaw 随包提供的官方模板，用户可以按需导入智能体。

这样可以避免两个问题：

- 用户刚打开 `智能体` 时，被大量官方玩法模板打扰，误以为这些都是自己项目里已经启用的智能体。
- `内置智能体` 的系统助手心智被海龟汤、狼人杀等娱乐应用稀释。

入口名称建议：

- 中文：`模板库`，副标题用 `官方模板`。
- 英文：`Template Library` 或 `Agent Templates`。

不建议只叫“示例”，因为传播感较弱；也不建议把页面叫“内置智能体”，因为这会和系统通用助手混淆。

## 列表结构

第一版不需要做复杂市场，只需要做一个清晰的模板库页面。

每个模板卡片展示：

- 应用名称
- 一句话结果承诺
- 标签：游戏、创作、办公、图片、知识、产品、社交等（后续可展示）
- 推荐输入示例
- 导入智能体按钮
- 导入并打开按钮
- 查看说明 / 查看源码入口

模板默认不进入 `/workflows/<id>/chat`，因为它还不是当前项目的工作流。用户点击 `导入智能体` 后，系统将模板复制到当前项目目录，再返回导入结果：

- `imported: true`
- `target_dir`
- `workflow_id`
- `message`
- 是否需要重启或热加载

如果当前运行时支持热加载，则导入后可直接跳转到新工作流聊天页；如果不支持，则明确提示用户重启服务后在 `智能体` 列表中打开。

## 内置目录约定

Claw Apps 模板不应放在项目根目录下的外部示例文件夹中，而应作为 AgentClaw 包的一部分发布。对外产品名统一使用“模板库”；内部实现目录名可以沿用 `agent_square`，但不应作为 UI 入口名称：

```text
agentclaw/
  agent_square/
    __init__.py
    <app_id>/
      claw_app.json
      README.md
      agents/
        __init__.py
        <workflow>.py
      data/
        ...
```

约定说明：

- `agentclaw/agent_square/__init__.py` 提供轻量 registry，用于列出模板库内容，避免 Dashboard 直接依赖文件系统细节。
- `agent_square` 不是独立可运行项目目录，不放每个 App 自己的 `server.py`、`.env.example` 或 `models.json.example`；它只承载 AgentClaw 包内置应用的 manifest、工作流代码和素材。
- 每个 App 必须提供 `claw_app.json`，描述 `id`、名称、分类、标签、入口工作流文件、`workflow_id`、推荐输入等展示信息。
- App 的核心工作流必须位于该 App 目录内，但不应在 AgentClaw 服务启动时自动导入注册。
- Dashboard 的智能体列表不应通过 `include_builtin=true` 把模板库模板混入用户智能体。模板库内容应通过单独 API 展示。
- 模板库 API 只读取 manifest，不执行模板代码，避免模板在用户未选择前污染运行时注册表。
- 导入 API 负责把模板复制到当前项目目录，例如 `<project>/agents/<app_id>/` 或 `<project>/agent_square/<app_id>/`。复制后，该模板才进入用户项目资产边界。
- 打包时需要把 `agent_square/**/*` 纳入 `agentclaw` package data，确保 pip 安装后仍能看到内置应用。
- 根目录不应再出现独立的 `agent_square/`，否则会让内置应用和外部示例边界变得混乱。

## 导入语义

模板库的核心动作是“导入智能体”，不是“直接运行”。

推荐接口：

- `GET /admin/dashboard/template-library/apps`：列出官方模板，只读取 `claw_app.json`。
- `POST /admin/dashboard/template-library/apps/{id}/import`：将模板导入到当前项目。

导入时应遵循：

- 不覆盖用户已有文件，除非用户明确确认覆盖。
- 如果目标 workflow_id 已存在，返回可理解的冲突信息，并建议用户改名或删除旧版本。
- 导入结果必须包含目标目录，方便用户知道模板代码被放到了哪里。
- 模板导入后应作为用户项目工作流对待，不再带 `is_builtin=true`。
- 后续可以扩展“创建副本名称”“选择目标目录”“导入后打开配置页”等能力。

## 迁移原则

当前已经实现的海龟汤工作流应作为第一批模板保留，但需要调整暴露方式：

- 从 `智能体` 列表移除未导入的海龟汤。
- 从 `内置智能体` 心智中移除海龟汤。
- 在 `模板库` 中展示海龟汤模板卡片。
- 用户点击 `导入智能体` 后，再把海龟汤复制到项目目录并注册为普通工作流。
- 已经导入到项目里的海龟汤，表现应和用户自己写的工作流一致，可以聊天、查看详情、配置、调试和分享。

## 第一批工作流选择原则

第一批不追求数量，追求“可玩、可截图、可分享、能体现 AgentClaw 能力”。

优先选择：

- 输入门槛低：一句话或少量文本就能开始。
- 结果强感知：故事、图片、角色、报告、游戏局势等容易理解。
- 多轮互动明显：能展示 AgentClaw 的对话和状态管理能力。
- 可解释为工作流：背后有节点、路由、工具或 Skill，不只是简单聊天。
- 失败成本低：模型偶尔不完美也不会造成严重问题。

避免第一批选择：

- 需要大量私有配置的企业流程。
- 依赖复杂外部账号授权的工具链。
- 结果不直观的纯工程类任务。
- 太像普通 ChatGPT prompt 的单节点问答。

## 推荐内置工作流清单

### 1. 海龟汤主持人

一句话：用户提问，AI 只回答“是 / 不是 / 无关”，逐步引导用户猜出离奇故事真相。

核心玩法：

- 系统先生成或选择一个海龟汤谜题，隐藏汤底。
- 用户通过自然语言提问。
- AI 根据汤底只回答“是”“不是”“无关”“接近了”。
- 当用户猜到关键真相，AI 揭示完整故事并给出推理路径。

适合展示的 AgentClaw 能力：

- 多轮对话状态管理。
- 隐藏信息保护。
- 规则约束输出。
- 游戏进度判断。
- 可配置谜题库或动态生成谜题。

传播点：

- 很适合录屏和直播。
- 用户天然愿意继续追问。
- 可以生成“今日离奇汤面”作为分享内容。

设计注意：

- AI 不能提前泄露汤底。
- 回答必须短，保持主持人感。
- 要有“放弃并看答案”“给我一点提示”的机制。
- 谜题应避免血腥、低俗、现实伤害等不适内容。

### 2. AI 狼人杀

一句话：用户加入一局由多个 AI 角色参与的狼人杀，AI 扮演玩家、主持人和推理对手。

核心玩法：

- 用户选择身份：村民、狼人、预言家、女巫、猎人等。
- AI 扮演主持人和其他玩家。
- 夜晚阶段自动推进角色行动。
- 白天阶段 AI 玩家发言、互相怀疑、投票。
- 用户可以发言、投票、伪装、带节奏。

适合展示的 AgentClaw 能力：

- 多 Agent 角色模拟。
- 状态机和阶段路由。
- 隐藏身份和私有信息控制。
- 结构化投票与胜负判断。
- 长上下文和角色一致性。

传播点：

- “和 7 个 AI 玩狼人杀”很容易吸引点击。
- 每局都有戏剧性，适合截图传播。
- 能体现 AgentClaw 不是普通聊天，而是可编排多角色系统。

设计注意：

- 第一版建议做轻量规则，避免完整狼人杀过复杂。
- AI 发言要短，有个性，不要长篇分析。
- 用户身份可以先固定为村民或狼人，降低实现难度。
- 要提供“快速局”：5-8 分钟结束。

### 3. AI 跑团主持人

一句话：用户进入一个文字冒险世界，AI 担任地下城主持人，生成场景、事件、检定和结局。

核心玩法：

- 用户选择世界观：赛博朋克、奇幻、末日、侦探、校园怪谈等。
- AI 生成角色卡和开场场景。
- 用户自由行动。
- 关键动作触发随机检定或属性判定。
- AI 维护背包、生命值、关系和任务线。

适合展示的 AgentClaw 能力：

- 长程状态维护。
- 工具节点生成随机数。
- 分支剧情路由。
- 结构化角色数据。
- 多轮叙事生成。

传播点：

- “一分钟开一局 AI 跑团”有强娱乐属性。
- 用户很容易分享自己的离奇结局。
- 可作为 AgentClaw 的“沉浸式工作流”代表。

设计注意：

- 第一版不要做复杂规则书。
- 保持轻量属性：生命、理智、金币、背包即可。
- 每 5-8 轮给一个阶段性结局，避免无限拖长。

### 4. AI 侦探案发现场

一句话：用户作为侦探，通过询问嫌疑人、查看线索和推理，找出真凶。

核心玩法：

- AI 生成案件背景、嫌疑人、时间线和关键证据。
- 用户可以询问不同嫌疑人。
- 每个嫌疑人有自己的口吻、动机和隐瞒点。
- 用户可以申请查看线索。
- 最后提交凶手、动机、作案手法。

适合展示的 AgentClaw 能力：

- 多角色模拟。
- 隐藏真相和线索逐步释放。
- 用户输入路由到不同角色。
- 最终答案评分。

传播点：

- 比普通聊天更像游戏。
- 可以产出“案件卡片”和“推理报告”。
- 适合做短视频演示。

设计注意：

- 案件内容要偏轻推理，避免真实暴力细节。
- 嫌疑人发言要短且有差异。
- 系统需要约束不能随意改真相。

### 5. 小红书 / 推文内容工坊

一句话：输入一个想法，AI 生成多平台内容包，包括标题、正文、标签、封面提示词和发布建议。

核心玩法：

- 用户输入主题或产品。
- AI 判断平台风格：小红书、微博、X、LinkedIn、公众号。
- 生成多版标题和正文。
- 可调用图片生成 skill 生成封面或配图。
- 输出发布日历和 A/B 测试建议。

适合展示的 AgentClaw 能力：

- 多节点内容生成。
- 风格路由。
- 图片生成集成。
- 结构化输出。

传播点：

- 非开发者马上能用。
- 结果适合截图。
- 和图片生成功能天然结合。

设计注意：

- 不要过度承诺涨粉。
- 需要避免生成虚假背书和违规营销内容。

### 6. AI 海报 / 封面生成器

一句话：输入主题，AI 生成视觉方向、文案、提示词，并调用图片模型产出海报或封面。

核心玩法：

- 用户输入活动、产品、文章或视频主题。
- AI 先生成风格方向和文案。
- 用户确认或选择风格。
- 图片生成节点生成封面。
- 输出可复制的标题、描述和图片文件。

适合展示的 AgentClaw 能力：

- 多步生成。
- Human-in-the-loop 确认。
- 图片生成 provider 选择。
- 文件输出和浏览器可展示 URL。

传播点：

- 视觉冲击最强。
- 很适合 README 首屏 GIF。
- 能展示新加入的 image-generation skill。

设计注意：

- 缺少图片 API Key 时要优雅提示配置。
- 文本和图片分两步，降低一次生成失败率。

### 7. 产品命名与品牌工坊

一句话：输入产品想法，AI 输出品牌名、slogan、定位、Logo 方向、配色和首屏文案。

核心玩法：

- 用户描述产品。
- AI 生成多个命名方向。
- 用户选择喜欢的方向。
- AI 继续生成品牌故事、视觉关键词、Logo prompt、落地页首屏文案。

适合展示的 AgentClaw 能力：

- 多轮偏好收敛。
- 多候选方案比较。
- 结构化品牌资产输出。
- 可选图片生成。

传播点：

- 对独立开发者、创业者、设计师都友好。
- 很容易出“我的产品被 AI 包装了一遍”的效果。

### 8. 简历 / 作品集改造师

一句话：上传简历或粘贴经历，AI 生成更好的简历表达、项目亮点和面试问答。

核心玩法：

- 用户上传或粘贴简历。
- AI 提取经历结构。
- 生成问题诊断和修改建议。
- 输出新版简历片段、项目 STAR 表达、面试自我介绍。

适合展示的 AgentClaw 能力：

- 文档解析。
- 结构化抽取。
- 多阶段改写。
- 可导出结果。

传播点：

- 实用性强。
- 非开发者也懂。
- 容易带来真实使用。

设计注意：

- 隐私提示必须明显。
- 不应伪造经历，只优化表达。

### 9. 会议纪要到行动计划

一句话：输入会议记录，AI 自动提炼结论、待办、负责人、风险和后续邮件。

核心玩法：

- 用户粘贴会议文本或上传文档。
- AI 提取议题、决策和行动项。
- 生成任务表格。
- 生成后续跟进邮件或群消息。

适合展示的 AgentClaw 能力：

- 文档处理。
- 结构化输出。
- 多格式结果。
- 后续渠道发送能力的扩展空间。

传播点：

- 实用但不够娱乐，适合作为第二批或办公类代表。

### 10. 文章变播客脚本

一句话：输入文章或链接，AI 改写成双人播客脚本、开场白、分段和金句。

核心玩法：

- 用户输入文章。
- AI 提取核心观点。
- 生成两位主持人的自然对话。
- 输出标题、简介、分段时间线、口播脚本。

适合展示的 AgentClaw 能力：

- 长文理解。
- 风格化改写。
- 多角色文本生成。
- 后续可扩展 TTS。

传播点：

- 内容创作者友好。
- 结果形式新鲜。

### 11. AI 塔罗 / 情绪占卜

一句话：用户输入最近困惑，AI 抽牌并给出轻量心理建议。

核心玩法：

- 用户输入问题。
- 工具节点随机抽取牌面。
- AI 解读牌面，但以陪伴和反思为主。
- 输出行动建议和一句鼓励。

适合展示的 AgentClaw 能力：

- 工具随机性。
- 结构化流程。
- 情感化输出。

传播点：

- 非开发者非常容易上手。
- 适合社交传播。

设计注意：

- 明确娱乐性质。
- 避免医疗、投资、法律等高风险建议。

### 12. 今日灵感盲盒

一句话：每天生成一个创作挑战、一个角色设定、一张视觉提示词和一个行动任务。

核心玩法：

- 用户点击打开盲盒。
- AI 生成今日主题。
- 可选择“写作、绘画、产品、学习、健身”等方向。
- 输出挑战卡片和可分享图片。

适合展示的 AgentClaw 能力：

- 定时任务。
- 图片生成。
- 个性化偏好。
- 可分享卡片。

传播点：

- 非常适合首页展示。
- 可以成为用户每天打开 AgentClaw 的理由。

## MVP 推荐组合

第一版建议只做 6 个，覆盖娱乐、创作、实用三类：

1. 海龟汤主持人
2. AI 狼人杀

## 工作流实现分层建议

每个 Claw App 都应该遵循同一套结构，便于复制和学习：

1. App 元数据
   - 名称、描述、标签、预览图、推荐输入。

2. 输入 schema
   - 必填输入尽量少。
   - 可选参数放到高级设置里。

3. 工作流定义
   - 用 AgentClaw 的真实 Workflow 实现，不做假 demo。
   - 尽量包含 2-5 个节点，展示编排价值。

4. 结果 schema
   - 输出 Markdown、JSON、图片文件、任务表等可统一渲染的结果。

5. 可复制能力
   - 复制为项目工作流。
   - 查看源码或工作流结构。
   - 用自己的模型和 API Key 运行。

## 风险与约束

### 不要变成玩具合集

趣味应用负责吸引力，但要始终连接回 AgentClaw 的核心能力。每个 App 都应该能被查看工作流结构，并能被复制为自己的工作流。

### 不要让配置门槛破坏首次体验

图片生成、搜索、外部工具等需要 API Key。第一版应该让纯文本工作流先能跑；依赖额外 key 的工作流应在入口明确标记，并在缺失时提供友好配置引导。

### 不要一开始做模板市场

市场需要供给和社区。第一阶段应做官方精选模板库，保证质量和传播效果。等官方 App 足够稳定，再考虑用户发布和社区模板。

### 不要让游戏工作流失控

狼人杀、跑团、侦探等涉及隐藏信息和角色扮演，需要明确规则、状态和边界。第一版应降低规则复杂度，避免模型随意改设定。

## 成功指标

可以用以下指标判断方向是否有效：

- GitHub README 首屏停留时间提升。
- Star 转化率提升。
- 用户第一次启动后运行内置 App 的比例。
- Claw Apps 的平均运行次数。
- “复制为我的工作流”的点击率。
- 海报、海龟汤、狼人杀等 App 的分享截图数量。
- Issues / Discussions 中围绕 App 扩展的请求数量。

## 建议路线

### 第一阶段：官方精选 MVP

- 新增“模板库”入口。
- 模板库内容不默认注册到用户工作流列表。
- 支持把官方模板导入为当前项目智能体。
- 上线 6 个官方 App。
- 每个 App 支持查看说明、导入智能体、导入后运行和查看工作流。
- README 增加 GIF 展示。

### 第二阶段：增强沉浸感

- 为海龟汤、狼人杀、跑团增加更强的游戏状态 UI。
- 为图片类和内容类 App 增加结果卡片渲染。
- 支持收藏、最近运行和示例输入。

### 第三阶段：模板化与社区化

- 支持用户把自己的工作流发布为本地 App。
- 支持导出 App 包。
- 支持 ClawHub 或类似机制安装更多官方/社区 App。

## 总结

内置趣味工作流不是简单增加 demo，而是给 AgentClaw 增加一个面向普通用户的体验层、面向开发者的教学层、面向传播的展示层。

AgentClaw 现在已经有“能力”，下一步需要让人第一眼感到“这个东西好玩、能用、值得收藏”。Claw Apps 是一个合适的承载方式。第一批应以海龟汤、AI 狼人杀、AI 海报生成器等高传播工作流打开局面，再用内容工坊、品牌工坊、会议纪要等实用工作流证明它不是玩具，而是一个可以持续扩展的 AI 工作台。
