# AgentClaw Examples

This directory is a runnable AgentClaw project containing a collection of example workflows. It follows the same project shape generated by `agentclaw init`: workflows live under `agents/`, `server.py` imports them, and the project is started with `agentclaw up` or `agentclaw serve`.

## Quick Start

```bash
cd agentclaw/examples
cp models.json.example models.json
# Edit models.json and fill in your model API keys / base URLs.

# Full startup wizard: choose Docker or Remote mode.
agentclaw up

# Or only start the server for an already configured local/remote environment.
agentclaw serve
```

Then open `http://localhost:8000/dashboard` and run any workflow from the Dashboard.

For API calls, use the unified endpoint:

```bash
curl -X POST http://localhost:8000/api/workflow/run \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <WORKFLOW_API_KEY>" \
  -d '{"workflow_id": "hello_world", "inputs": {"user_input": "Hello!"}}'
```

If `WORKFLOW_API_KEY` is not configured, AgentClaw generates a temporary key at startup and prints it in the server logs.


| No. | Workflow ID | File | Description | Key Concepts |

| 01 | `hello_world` | `agents/hello_world.py` | Basic LLM chat | `Workflow`, `LLMNode`, `Input`, streaming, multi-turn `thread_id` |

## Notes

- `models.json` is intentionally not committed. Copy `models.json.example` and fill in real credentials locally.
- `.env.example` contains optional runtime overrides. `agentclaw up` also works without copying it; missing runtime auth keys are generated at startup, and `MCP_TOKEN` is persisted to the project `.env`.
- Some examples need optional infrastructure or dependencies:
  - `doc_analyzer`: install document parsing support, for example `pip install agentclaw-ai[document]`.
  - `kb_rag`: configure PostgreSQL, Milvus, embedding/rerank models, and upload documents through the Dashboard.
  - `mcp_agent`: uses `mcps/example_tools.py`, which publishes deterministic local tools into the running AgentClaw service, and `mcp.json`, which configures the external `fetch` MCP server. Remote clients should connect to `GET /mcp/examples-tools/sse` with `Authorization: Bearer <MCP_TOKEN>`; if `MCP_TOKEN` is missing, AgentClaw generates one and writes it to the project `.env`.
