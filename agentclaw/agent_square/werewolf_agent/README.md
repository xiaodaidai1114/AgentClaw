# AI 狼人杀智能体

这是 AgentClaw 内置 Claw Apps / 智能体广场中的官方工作流应用。当前版本重点展示复杂多角色智能体的推荐架构：主 `workflow` 维护狼人杀主状态机，12 个座位通过同一个角色子工作流实例化，并由 `SubWorkflowNode` 完成状态隔离、输入视角映射、输出回写和追踪串联。

这个目录随 `agentclaw` 包发布，设计目标不是外部 demo 项目，而是 Dashboard “智能体”列表中可筛选、可运行、可查看实现、可复制改造的内置 App。代码重点示范复杂游戏如何拆成“主控状态机 + 多个角色子工作流”，后续可以在保持主图结构不变的前提下继续补全完整狼人杀规则。

## 默认板子

固定 12 人局：

- 狼人阵营：3 普通狼人、1 狼王
- 好人阵营：预言家、女巫、猎人、守卫、白痴、3 平民

当前已落地的主要流程：

- 用户随机身份，进入首夜。
- 首夜按 `狼人 -> 预言家 -> 守卫 -> 女巫 -> 猎人` 的主状态机顺序推进。
- 当用户身份属于当前夜间阶段时，主工作流暂停并只向用户展示自然语言行动提示。
- 当前阶段不属于用户时，主工作流只调用该阶段需要行动的 AI 角色子工作流。
- 首夜天亮后先进入首日警长竞选，警长竞选结束后再公布首夜死讯。
- 用户选择上警或不上警后，主工作流并行调用所有 AI 角色子工作流决定是否上警。
- 主工作流生成警上名单和警下名单，并按随机起点后的警上座位顺序推进发言。
- 警下投票、警长当选、警长 PK、无警长分支、警徽移交均由主状态机处理。
- 首夜/白天死亡会进入统一死亡中断链：警徽移交、猎人/狼王开枪、遗言、胜负判定。
- 白天由警长决定发言起点，警长最后归票；无警长时按座位顺序发言。
- 放逐投票支持投票摘要、平票 PK、再次平票无人出局、白痴翻牌免死。
- 狼人/狼王在发言阶段可通过按钮或文本自爆，白天立即结束并进入黑夜。
- 预言家查验会进入私有视角；狼人玩家会看到狼队友；结束后统一身份复盘。
- 默认发布工作流启用模型角色子工作流；如果运行环境没有 LLMManager，会自动降级到本地兜底决策，便于导入和测试。

仍建议继续完善的方向：

- 让模型角色的长期记忆更会总结站边、票型和关键矛盾。
- 扩展更多板子和可配置规则，例如是否死亡翻牌、是否允许女巫首夜自救。
- 增加端到端模拟局回放测试，覆盖多日夜连锁胜负边界。

## 运行

```bash
agentclaw up --mode remote
```

打开 Dashboard 后进入 `智能体`，将筛选切到 `内置广场`，选择 `AI 狼人杀主持人`。AgentClaw 服务启动时会自动注册 `agentclaw/agent_square` 下的内置工作流。

## 建议对话

```text
使用默认规则开始。
```

游戏过程中根据身份输入：

```text
上警
不上警
我发言：4号的起跳不像真预言家。
```

## 设计约束

- 完整身份表只存在于内部 session，不直接传给普通 AI 节点或玩家。
- 主工作流是唯一的流程状态机；角色子工作流只产出 `action` 或 `speech`。
- 每个 AI 座位使用派生 thread id，例如 `game-thread:actor:p7`，便于日志、追踪和可视化定位。
- 每个 AI 座位有自己的 `actors.pX.memory`，通过 `state_map` 映射进出子工作流。
- 子工作流通过 `readonly_input_map` 接收当前视角和请求，通过 `output_map` 回写 `actor_outputs.pX`。
- 当前阶段只向用户展示自然语言提示，不暴露内部 action 字段。
